# Server.js — Reference & Glossary

This document explains **server.js**: the terms it uses, what each variable/function does, and how each socket event works. Use it when you need to quickly see what something does.

---

## 1. Key terms (glossary)

| Term | Meaning |
|------|--------|
| **timerId** | Unique ID of a timer (e.g. `"abc123"`). Generated by `generateId()`. Used to find the timer in `timers` and to name its Socket.IO room. |
| **controllerId** | Identity of the person who owns timers. Stored in a cookie on the client. One controller can have many timers and can have multiple tabs (each tab = one socket, same controllerId). |
| **socket** | One WebSocket connection (one browser tab or device). Has a unique `socket.id` on the server. |
| **room** | A Socket.IO group. Everyone in a room receives broadcasts to that room. We use one room per timer: room name = `"timer:" + timerId`. |
| **timer** | One countdown instance: name, duration, running state, styling, etc. Stored as a `Timer` class instance in the `timers` Map. |
| **viewer** | Someone watching a timer (viewer page). They join the timer’s room via `view-timer` or `join-timer`. |
| **controller** | The person who created/owns the timer. They control start/pause/reset and see the “controller” UI. |

---

## 2. Global data (what the server remembers)

| Variable | Type | What it stores |
|----------|------|----------------|
| **timers** | `Map` | `timerId` → `Timer` instance. All existing timers. |
| **controllerTimers** | `Map` | `controllerId` → Set of `timerId`. Which timers each controller owns. |
| **controllerToSocket** | `Map` | `controllerId` → Set of `socket.id`. Which sockets belong to that controller (e.g. their open tabs). Used to send “timer list” or “limit exceeded” to all of a controller’s tabs. |

---

## 3. Helper functions (top-level)

### `timerRoomId(timerId)`
- **What it does:** Returns the Socket.IO room name for that timer.
- **Returns:** `"timer:" + timerId` (e.g. `"timer:abc123"`).
- **Why:** So we always use the same room name for join, leave, and broadcast.

### `generateId()`
- **What it does:** Creates a new unique ID for a timer.
- **Returns:** A string like `"m5k2x9"` (time-based + random).
- **Used when:** Creating a new timer.

### `getTimersForController(controllerId)`
- **What it does:** Builds the list of timers that this controller owns, with current viewer count.
- **Returns:** Array of `{ id, name, duration, connectedCount, joiningCode }`. `connectedCount` comes from the room size (how many sockets are in that timer’s room).
- **Used by:** Any code that needs to send the “your timers” list to the client.

### `emitTimerListForController(socket, controllerId)`
- **What it does:** Sends the “your timers” list to **one** socket (the one you pass in).
- **How:** Calls `getTimersForController(controllerId)` and emits `'timer-list'` to that socket.
- **When to use:** After an action from that same socket (e.g. create timer, delete timer, get-timers, or any control action) so that tab gets an updated list.

## 4. Timer class (one countdown instance)

Each timer is an object with state and methods. All “business logic” for one countdown lives here.

### Properties (what a timer stores)

| Property | Meaning |
|----------|--------|
| id | Same as timerId. |
| name | Display name. |
| duration | Total seconds (can change when user “set” or “adjust”). |
| originalDuration | Duration when last set/reset (used for Reset). |
| remaining | Seconds left (updates every second when running). |
| isRunning | Whether the countdown is currently running. |
| startTime | When it was started (used to compute remaining). |
| message | On-screen message (e.g. “Break time”). |
| backgroundColor, textColor, fontSize, timerView | Styling. |
| isFlashing | Whether the display is flashing. |
| joiningCode | Short code for viewers to join. |
| maxConnectionsAllowed | Max viewers (from plan). |
| maxTimersAllowed | Max timers per controller (from plan). |
| controllerId | Who owns this timer. |
| interval | Node.js interval for ticking every second when running. |

### Methods (what you can do to a timer)

| Method | What it does |
|--------|----------------|
| **start()** | Starts the countdown (sets isRunning, startTime, and an interval that calls update() every second). |
| **pause()** | Stops the countdown and saves current remaining time. |
| **reset()** | Stops and sets duration/remaining back to originalDuration. |
| **setDuration(duration)** | Sets total and remaining to this value and stops the timer. |
| **adjustTime(seconds)** | Adds/subtracts seconds from duration (and remaining if running). |
| **update()** | Recomputes remaining if running, then sends current state to everyone in this timer’s room via `timer-update`. |
| **updateMessage(message)** | Sets the on-screen message and broadcasts. |
| **clearMessage()** | Clears the message and broadcasts. |
| **updateStyling(styling)** | Updates colors/font/view and broadcasts. |
| **toggleFlash(isFlashing)** | Turns flash on/off and broadcasts. |
| **updatejoiningCode(joiningCode)** | Sets the join code and broadcasts. |
| **getState()** | Returns a plain object with id, name, duration, remaining, isRunning, message, styling, connectedCount (from room size), joiningCode, etc. Used when sending state to the client. |

## 5. HTTP routes

### `GET /health`
- **What it does:** Simple health check.
- **Returns:** `{ status: "ok", timers: number, connectedSockets: number }`.
- **Use:** Load balancers or monitoring to see if the server is up and how many timers/sockets exist.

## 6. Socket events (client → server)

For each event we list: **what the client sends**, **what the server does**, and **what the server sends back** (to whom).

---

### `join-timer`
- **Meaning:** “Put this socket in this timer’s room (controller joining their own timer).”
- **Payload:** `{ timerId, controllerId, maxConnectionsAllowed }`
- **Server:**
  - Finds the timer; if missing, emits `timer-not-found` and stops.
  - Puts the socket in the timer’s room.
  - If room size ≥ maxConnectionsAllowed: removes socket from room, emits `timer-full` to that socket, emits `limit-exceeded` to the controller’s sockets, stops.
  - Otherwise: adds this socket to controllerToSocket for that controllerId, sends `timer-joined` + full timer state to this socket, calls `timer.update()` (so everyone in room gets latest state). If this socket is the timer’s owner, also sends updated timer list to this socket via `emitTimerListForController`.
- **Sent to client:** `timer-joined` (state), or `timer-not-found`, or `timer-full`; controller may get `limit-exceeded` and `timer-list`.

---

### `view-timer`
- **Meaning:** “Put this socket in this timer’s room (viewer opening the timer).”
- **Payload:** `{ timerId, controllerId, maxConnectionsAllowed }`
- **Server:** Same as join-timer (room join, capacity check, controllerToSocket, `timer-joined`, `timer.update()`). Does **not** send the timer list to this socket (viewers don’t need the list).
- **Sent to client:** `timer-joined` or `timer-not-found` or `timer-full`; controller may get `limit-exceeded`.

---

### `create-timer`
- **Meaning:** “Create a new timer and put this socket in its room.”
- **Payload:** `{ name, duration, maxConnectionsAllowed, maxTimersAllowed, controllerId, styling }`
- **Server:**
  - If this controller already has ≥ maxTimersAllowed timers, emits `limit-exceeded` (type: timers) and stops.
  - Creates a new Timer, sets controllerId, applies styling, stores it in `timers` and `controllerTimers`, puts this socket in the new timer’s room, adds socket to controllerToSocket.
  - Sends updated timer list to this socket and sends `timer-created` with full state.
- **Sent to client:** `timer-created` (state) + `timer-list`, or `limit-exceeded`.

---

### `get-timers`
- **Meaning:** “Send me the list of timers I own.”
- **Payload:** `{ controllerId }`
- **Server:** Calls `emitTimerListForController(socket, controllerId)` so this socket gets the list.
- **Sent to client:** `timer-list` (array of timers with id, name, duration, connectedCount, joiningCode).

---

### `leave-timer`
- **Meaning:** “Remove this socket from this timer’s room (e.g. user switched timer or navigated away).”
- **Payload:** `{ timerId }`
- **Server:** If the timer exists, removes this socket from that timer’s room and calls `timer.update()` so everyone left in the room gets the new state (and connectedCount).
- **Sent to client:** No direct reply; others in the room get `timer-update` with updated count.

---

### `delete-timer`
- **Meaning:** “Delete this timer (only if I am the owner).”
- **Payload:** `{ timerId, controllerId }`
- **Server:** If the timer exists and its controllerId matches, broadcasts `timer-deleted` to the timer’s room, clears the timer’s interval, removes it from `timers` and `controllerTimers`, then sends updated timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; everyone in the room gets `timer-deleted`.

---

### `set-timer`
- **Meaning:** “Set this timer’s duration (and stop it).”
- **Payload:** `{ timerId, duration, controllerId }`
- **Server:** If the timer exists and this socket is the owner, calls `timer.setDuration(duration)` then sends updated timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; everyone in the room gets `timer-update` (from Timer.update() inside setDuration).

---

### `start-timer`
- **Meaning:** “Start the countdown.”
- **Payload:** `{ timerId, controllerId }`
- **Server:** If owner, calls `timer.start()` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; everyone in the room gets `timer-update` every second while running.

---

### `pause-timer`
- **Meaning:** “Pause the countdown.”
- **Payload:** `{ timerId, controllerId }`
- **Server:** If owner, calls `timer.pause()` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `reset-timer`
- **Meaning:** “Reset the timer to its original duration and stop it.”
- **Payload:** `{ timerId, controllerId }`
- **Server:** If owner, calls `timer.reset()` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `adjust-timer`
- **Meaning:** “Add or subtract seconds from the timer.”
- **Payload:** `{ timerId, seconds, controllerId }`
- **Server:** If owner, calls `timer.adjustTime(seconds)` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `update-message`
- **Meaning:** “Set the on-screen message for this timer.”
- **Payload:** `{ timerId, message, controllerId }`
- **Server:** If owner, calls `timer.updateMessage(message)` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `clear-message`
- **Meaning:** “Clear the on-screen message.”
- **Payload:** `{ timerId, controllerId }`
- **Server:** If owner, calls `timer.clearMessage()` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `update-styling`
- **Meaning:** “Change colors/font/view for this timer.”
- **Payload:** `{ timerId, styling, controllerId }`
- **Server:** If owner, calls `timer.updateStyling(styling)` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `toggle-flash`
- **Meaning:** “Turn display flash on or off.”
- **Payload:** `{ timerId, isFlashing, controllerId }`
- **Server:** If owner, calls `timer.toggleFlash(isFlashing)` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `update-joining-code`
- **Meaning:** “Set the short code viewers use to join this timer.”
- **Payload:** `{ timerId, joiningCode, controllerId }`
- **Server:** If owner, calls `timer.updatejoiningCode(joiningCode)` then sends timer list to this socket.
- **Sent to client:** This socket gets `timer-list`; room gets `timer-update`.

---

### `disconnecting` (Socket.IO built-in)
- **Meaning:** Fires when the socket is about to disconnect; rooms are still available.
- **Server:** Saves the list of timer room IDs this socket was in (`socket._timerRoomIds`) so we can use it in `disconnect` (when rooms are already cleared).
- **Sent to client:** Nothing.

---

### `disconnect` (Socket.IO built-in)
- **Meaning:** This socket has disconnected (tab closed, navigated away, network lost).
- **Server:** Uses the saved timer room IDs; in a `setImmediate` (after Socket.IO has removed the socket from rooms), calls `timer.update()` for each of those timers so remaining viewers get updated count. Then removes this socket from `controllerToSocket`.
- **Sent to client:** Nothing (socket is gone). Other clients in the same timer room get `timer-update` with new connectedCount.

---

## 7. Events the server sends to the client

| Event | Who gets it | Meaning |
|-------|-------------|--------|
| **timer-list** | One or more sockets (controller’s tabs) | Array of timers (id, name, duration, connectedCount, joiningCode) for “Your Timers” list. |
| **timer-joined** | The socket that just joined | Full timer state (so the client can show the timer). |
| **timer-update** | Everyone in that timer’s room | Latest timer state (remaining, isRunning, message, styling, connectedCount, etc.). |
| **timer-created** | The socket that created it | Full state of the new timer. |
| **timer-deleted** | Everyone in that timer’s room | `{ timerId }` — this timer no longer exists. |
| **timer-not-found** | The socket that tried to join/view | `{ timerId }` — no such timer. |
| **timer-full** | The socket that tried to join | Room is at max capacity; they were not added. |
| **limit-exceeded** | Controller’s sockets | Plan limit hit (too many timers or too many viewers); message explains which. |

## 8. Quick lookup: “Where is …?”

| I want to … | Look at … |
|-------------|-----------|
| See how room names are built | `timerRoomId()` |
| See how we build the “your timers” list | `getTimersForController()` |
| Send the list to one socket | `emitTimerListForController(socket, controllerId)` |
| See what one timer’s state looks like | `Timer.getState()` |
| See how we broadcast to a timer’s viewers | `Timer.update()` → `io.to(roomId).emit("timer-update", ...)` |
| See what happens when someone joins a timer | `join-timer` or `view-timer` handler |
| See what happens when someone leaves a timer | `leave-timer` handler and `disconnect` handler |
| See what happens when the controller does an action | Any of the “TIMER CONTROLS” handlers (set, start, pause, reset, adjust, message, styling, flash, joining-code) |
